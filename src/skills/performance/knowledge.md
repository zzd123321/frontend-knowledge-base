# 前端性能优化核心知识点

前端性能优化不是单一技巧，而是围绕用户体验、加载链路、渲染链路、网络请求、资源体积、运行时执行和监控体系展开的系统工程。真正有价值的性能优化，不是背概念，而是定位瓶颈、解释原因、设计方案、验证收益。

---

## 1. 性能优化的目标

常见目标包括：

- 首屏更快
- 交互更流畅
- 页面更稳定
- 资源更省

---

## 2. Web Vitals

重点指标：

- LCP：最大内容绘制
- INP：交互到下一次绘制
- CLS：累计布局偏移

---

## 3. 性能优化维度

- 加载性能
- 渲染性能
- 运行时性能
- 资源性能
- 网络性能
- 构建性能
- 监控分析

---

## 4. 关键渲染路径

浏览器大致流程：

1. 解析 HTML 构建 DOM
2. 解析 CSS 构建 CSSOM
3. 合成渲染树
4. Layout
5. Paint
6. Composite

---

## 5. 首屏加载优化

常见方式：

- 减少首屏 JS
- 路由和组件懒加载
- 关键 CSS 优化
- 图片优化
- 预加载关键资源
- SSR / SSG
- CDN 与缓存

---

## 6. 图片优化

优化方式：

- 合适格式：`webp`、`avif`
- 控制尺寸
- 压缩质量
- 懒加载
- CDN

---

## 7. 字体优化

- 只加载必要字重
- 子集化
- `font-display: swap`
- 预加载关键字体

---

## 8. 缓存优化

常见缓存包括：

- 浏览器缓存
- CDN 缓存
- Service Worker 缓存

---

## 9. 资源压缩与传输优化

- Gzip
- Brotli
- HTTP/2
- CDN
- 合理拆包

---

## 10. JavaScript 执行优化

问题通常来自：

- 包体过大
- 大量同步计算
- 初始化逻辑过重
- 主线程阻塞

---

## 11. 长任务与主线程阻塞

常见来源：

- 大循环
- 大量 JSON 解析
- 重计算
- 同步布局抖动

优化方式：

- 拆分任务
- 分帧处理
- `requestIdleCallback`
- Worker

---

## 12. 回流与重绘

### 回流

布局重新计算，开销更大。

### 重绘

样式变化但不影响布局时重新绘制。

---

## 13. 动画优化

高频动画更适合：

- `transform`
- `opacity`

因为它们通常不会频繁触发布局。

---

## 14. 列表与渲染优化

常见方式：

- 虚拟列表
- 分页
- 无限滚动
- 避免无意义重渲染

---

## 15. 框架层优化

### React

- `memo`
- `useMemo`
- `useCallback`
- 状态就近管理

### Vue

- `computed`
- 组件拆分
- `v-once`
- 合理使用响应式

---

## 16. 网络请求优化

- 合并或按需拆分请求
- 避免瀑布依赖
- 请求去重
- 取消无效请求
- 合理缓存

---

## 17. 防抖与节流

### 防抖

适合搜索输入、resize。

### 节流

适合 scroll、mousemove。

---

## 18. Service Worker 与离线缓存

可实现：

- 离线访问
- 资源缓存
- 请求拦截

---

## 19. SSR、SSG 与 CSR

### CSR

前端接管渲染，首屏通常一般。

### SSR

服务端输出 HTML，首屏更快、SEO 更好。

### SSG

构建阶段生成静态页面，适合内容型站点。

---

## 20. 监控与指标采集

常见方式：

- Lighthouse
- Chrome DevTools
- Web Vitals
- Performance API
- 前端埋点平台

---

## 21. 包体积优化

- Tree Shaking
- 动态导入
- 按需引入组件库
- 替换重型依赖
- bundle 分析

---

## 22. 内存优化

常见问题：

- 事件未清理
- 定时器未清理
- 闭包长期持有大对象
- 大量缓存未释放

---

## 23. 性能优化方法论

推荐流程：

1. 明确痛点
2. 采集数据
3. 定位瓶颈
4. 实施优化
5. 验证收益
6. 建立监控
